# hy2

Personal server-side configuration for Hysteria2 + Xray (VLESS+Reality) plus a
lightweight web subscription / user-management panel.

Designed for one-shot deployment on a fresh Debian/Ubuntu VPS.

## What you get

- **Hysteria2** listening on `:443/udp` with Salamander obfs and UDP port hopping over `20000-40000/udp`.
- **Xray** VLESS + Reality on `:443/tcp` (primary) and `:8443/tcp` (backup), masquerading as `www.bing.com`.
- A Python subscription / admin panel (`subscription_service.py`) that serves per-user Clash YAML subscriptions and enforces per-user monthly quota + device limits.
- A once-per-minute `traffic_limiter.py` job that pulls hysteria traffic stats into `state/usage.json` and auto-kicks guests who exceed quota.
- Billing cycle resets automatically on the **21st** of each month.

## One-shot deploy

On a fresh server (root):

```bash
git clone git@github.com:<you>/hy2.git
cd hy2
cp .env.example .env
# edit .env and fill in every value (see comments in the file for how to generate each one)
vim .env
sudo ./deploy.sh
```

`deploy.sh` will:

1. Install the `hysteria` and `xray` binaries (official installers).
2. Render the config templates with your `.env` values.
3. Drop files into `/root/hysteria/`, `/usr/local/etc/xray/`, `/etc/systemd/system/`.
4. Generate a self-signed TLS cert for Hysteria if one is missing.
5. `systemctl daemon-reload`, enable and start every unit.

## After deploy

- **Admin panel**: `http://<server>:8080/admin` — on first visit you will be asked to set an admin password (stored hashed in `subscription_meta.json`).
- **Add a user** from the panel; a subscription URL of the form `http://<host>:8080/sub/<name>?token=<token>` will be shown.
- **Traffic panel**: `http://<server>:8080/panel/<user>?token=<token>` — per-user usage + device stats.

## Files NOT in git (by design)

These are either per-server secrets or runtime state — never commit them:

- `.env` — real secret values
- `server.crt`, `server.key` — TLS cert (auto-generated by `deploy.sh`)
- `users.json` — user roster with password hashes & `sub_token`s
- `subscription_meta.json` — admin password hash
- `state/` — usage counters, online snapshot, reset log
